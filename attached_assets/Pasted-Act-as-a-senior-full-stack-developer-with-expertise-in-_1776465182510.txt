Act as a senior full-stack developer with expertise in Firebase.

I want to build a secure cloud-based platform to distribute my own Lua scripts using a licensing/key system, fully hosted on Firebase (free tier compatible).

The system must be designed around Firebase services and limitations.

-----------------------------------
CORE REQUIREMENTS
-----------------------------------

1. Admin Dashboard (React + Firebase Hosting)
- Admin login using Firebase Auth
- Protected routes (only admin access)
- UI to:
  - Generate license keys
  - Revoke / ban keys
  - Set expiration dates
  - View logs (script usage, IP, HWID)
  - Upload Lua scripts

2. Key System API (Firebase Cloud Functions)
- HTTPS callable or REST endpoints
- Validate license keys
- Bind key to HWID (first use)
- Reject if:
  - key invalid
  - expired
  - revoked
  - HWID mismatch
- Return structured JSON response

3. Script Storage & Delivery
- Store encrypted Lua scripts in Firebase Storage
- Store metadata (IV, auth tag, version) in Firestore
- Scripts must NEVER be stored in plaintext
- On request:
  - Validate key
  - Download encrypted script
  - Decrypt in memory only
  - Add watermark (license key or user ID)
  - Return script to client

4. Security Requirements
- Use AES-256-GCM encryption
- Store encryption key in Firebase environment config (not in code)
- Enforce HTTPS only
- Prevent direct public access to scripts in storage
- Implement basic rate limiting inside Cloud Functions
- Log all access attempts

5. Database Design (Firestore)
Design collections for:
- licenses
- scripts
- logs

Include fields like:
- key, hwid, expiresAt, revoked
- script storage path, IV, auth tag
- request logs (IP, timestamp, status)

6. Lua Client Loader
Provide a Lua script that:
- Sends license key + HWID to API
- Handles JSON response
- Requests script securely
- Executes returned code using loadstring
- Handles errors gracefully
